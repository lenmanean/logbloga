# Package Content Terminology Style Guide

## Standard Terms (Use These)

### Content Categories
- **Implementation Plan** (singular) — The main roadmap file
- **Platform Setup Guides** (plural) — Configuration guides for tools/services
- **Creative Decision Frameworks** (full name) — Worksheets for business decisions
- **Templates & Checklists** (full name) — Ready-to-use files and checklists
- **Launch & Marketing** (full name) — Launch and marketing guides
- **Troubleshooting** (singular) — Common issues and solutions
- **Time & Budget Planning** (full name) — Planning worksheets and guides

### UI Terms
- **Level 1, Level 2, Level 3** (not "Tier" or "Stage")
- **Package** (not "Bundle" or "Product" when referring to full package)
- **Content** (not "Materials" or "Resources")
- **File** (not "Document" or "Asset")

### Cost Terms
- **Platform costs** (not "Infrastructure costs" or "Service costs")
- **Expected revenue** (not "Potential earnings" or "Target income")
- **Time investment** (not "Time commitment" or "Duration")

### Feature Terms
- **AI leverage** (not "AI integration" or "AI usage") — Describes how AI accelerates work
- **Progress tracking** (not "Completion tracking") — User progress through content
- **Watermarking** (not "DRM" or "Copy protection") — Copyright protection feature

## Abbreviations
- MD (markdown files)
- PDF (portable document format)
- ZIP (compressed archives)
- UX (user experience)
- UI (user interface)
- DB (database)
- RLS (row-level security)

## Capitalization
- File extensions: lowercase (md, pdf, zip)
- Proper nouns: Standard caps (Next.js, Supabase, ChatGPT)
- Section headings: Title Case
